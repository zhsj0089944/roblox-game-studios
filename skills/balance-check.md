## Phase 1: 识别检查领域

从 `$ARGUMENTS[0]` 确定检查领域：

- **economy** → 积分/EP 经济系统、获取速率、消耗速率
- **combat** → PVP 伤害、暴击、闪避、击退
- **progression** → 升级曲线、进化树、解锁节奏
- **boss** → Boss 数值、伤害、血量、行为模式
- **loot** → 掉落概率、凋落物系统、水瓶/曲奇
- **field** → 力场系统、充能、时机倍率
- **file path given** → 直接加载该文件，从内容推断领域

如果没有参数，询问用户要检查哪个系统。

---

## Phase 2: 读取数据文件

读取 `GameConfig.lua` 和相关配置文件。记录所有读取的文件——它们会出现在报告的"数据来源"部分。

**关键配置文件**：
- `GameConfig.lua` — 主配置（积分公式、升级数值、进化树）
- `GameManager.server.lua` — 服务器逻辑（伤害计算、掉落逻辑）
- `PVPSystem.lua` — PVP 系统（伤害、暴击、闪避）
- `ForceFieldSystem.lua` — 力场系统
- `BossManager.lua` — Boss 数值
- `ObjectRefresher.lua` — 物体刷新、掉落逻辑

---

## Phase 3: 读取设计文档

读取 `DESIGN.md` 和相关设计文档，理解设计意图、调参范围、预期数值。

## Phase 4: 执行分析

### 经济系统检查

```lua
-- 积分公式验证
base × (1 + itemBonus) × (1 + multUp) × (1 + evoTouch) × eventMult × vipMult

-- 检查项：
-- 1. 各乘数范围是否合理（是否有乘数可以无限叠加）
-- 2. 极端情况（全满 buff 时积分是否溢出）
-- 3. EP 获取速率 vs 消耗速率
-- 4. 重置（炼化）回报率
```

**经济平衡检查清单**：
- [ ] 积分获取速率（新人 vs 老手）
- [ ] EP 获取速率 vs 进化树总消耗
- [ ] 升级费用曲线是否平滑
- [ ] 力场解锁费用是否合理
- [ ] 是否存在无限刷分漏洞

### 战斗系统检查

```lua
-- PVP 伤害公式
固定 20 伤害 + 暴击 ×1.5 = 30

-- 检查项：
-- 1. 暴击率（15%→30%）是否过高
-- 2. 闪避率（5%→15%）是否合理
-- 3. 击退力度（15→35 stud/s）是否影响游戏体验
-- 4. Boss 伤害 vs 玩家血量
```

**战斗平衡检查清单**：
- [ ] PVP TTK（击杀时间）是否合理
- [ ] 暴击/闪避的期望值
- [ ] Boss 难度曲线
- [ ] NPC 守卫数值

### 升级/进化系统检查

```lua
-- 升级系统：4 条线 × 5 级
-- 进化系统：6 棵树 × 5 级，总 EP 1860

-- 检查项：
-- 1. 升级费用曲线
-- 2. 进化树 EP 分配是否均匀
-- 3. 各进化路径的性价比
-- 4. 是否存在"必点"和"废点"
```

### 掉落系统检查

```lua
-- 凋落物三分支：
-- 35% 不掉 / 25% 水瓶(回血35HP) / 40% 曲奇(回体力)

-- 检查项：
-- 1. 掉落概率是否合理
-- 2. 水瓶回血量 vs 玩家血量
-- 3. 曲奇恢复量 vs 体力上限
-- 4. 吸附距离（8格）是否过大
```

## Phase 6: 修复与验证循环

展示报告后，使用 `AskUserQuestion`：
- 提示："平衡检查完成。下一步？"
- 选项：
  - `[A] 修复最高优先级问题 — 引导我完成`
  - `[B] 保存报告到 design/balance/balance-check-[system]-[date].md`
  - `[C] 到此为止 — 我会手动审查`

如果选 [A]：
- 询问要先处理哪个问题
- 引导用户更新 `GameConfig.lua` 或相关文件
- 每次修复后，提供重新运行相关检查的选项

如果选 [B]：
- 写入报告到 `design/balance/balance-check-[system]-[date].md`
- 确认文件已写入，结束提示："修复后重新运行 `/roblox-balance-check` 验证。"

## Roblox 特有检查项

### DataStore 限制
- [ ] 单次请求 < 4MB
- [ ] 请求频率限制（60s 内 /player）
- [ ] Session Lock 处理

### 性能影响
- [ ] 物体数量上限（1700）
- [ ] 热路径内存分配
- [ ] RemoteEvent 频率

### 服务器/客户端同步
- [ ] 数值计算在服务器
- [ ] 视觉效果在客户端
- [ ] 防作弊验证

## 常见问题模式

### 1. 乘数叠加失控
```lua
-- 问题：多个乘数可以无限叠加
base × mult1 × mult2 × mult3 × ...

-- 修复：限制最大乘数或使用加法叠加
base × min(mult1 + mult2 + mult3, MAX_MULT)
```

### 2. 经济通胀
```lua
-- 问题：获取速率 >> 消耗速率
-- 修复：增加消耗点或降低获取速率
```

### 3. 进度断层
```lua
-- 问题：某一级别突然需要大量资源
-- 修复：平滑费用曲线
```

### 4. 必点路径
```lua
-- 问题：某个进化/升级性价比远高于其他
-- 修复：调整数值使各路径平衡
```
